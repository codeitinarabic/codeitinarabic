ما هو ال Static Page Generator ولماذا نعيد استخدام ال Static Page مرة أخري؟

تطوير تطبيقات الويب 2019-01-29

#معلومة_ويب

الصفحات الثابتة أو ال Static Page هى تعتبر من أقدم تقنيات الويب بل هي اساس مواقع الانترنت منها بدأ الانترنت حيث لم يكن هناك كود في الخادم Server Side أو قواعد بيانات فضل عن كود أو برنامج في ال client side مجرد ملف يتم فتحه بالمتصفح ليعرض بيانات فقط أو ربما سكريبتات بسيطة بالجافا سكريبت

كان الامر كذلك وكان كافيا جدا لعرض المعلومات لكن سرعان ما تطور الامر ليظهر الاكواد التي تعمل في الخوادم للتتلقي البيانات من المستخدمين وتعيد ارسالها مرة اخري واصبح ذلك تطورا كبيرا فقد ظهر ما يسمي بالصفحات التفاعلية حيث ان الموقع اصبح يعرض بيانات للمستخدم ويتلقي من المستخدم بيانات ايضا ويخزن البيانات هذة في قواعد البيانات التي تعمل في الخادم ... لم يتوقف الامر علي هذا بل ظهرت ما يسمي بال front end webapp او الاكواد والتطبيقات التي تعمل علي جانب المستخدم لتتفاعل معة سريعا Real-Time App قبل الرجوع الي الخادم وبالتالي زياده في التفاعلية مع المستخدم وسرعه أكبر.

ولكن وبعد كل هذا لماذا نريد العوده للعمل بال static page مرة أخري وهل من الممكن خلق تفاعل مع المستخدم من خلال Static Page ؟

لا يخفي علي اي مطور المميزات الموجوده بال static website ولعل اشهرها :-

1- السرعة الفائقة للموقع

2- نسبة أمان أعلي من ال Dynamic Web Page

3- صديق محرك البحث SEO Friendly حيث تستطيع محركات البحث التعرف عليها وفهرستها بسهولة بخلاف ال Dynamic و ال PWA نظرا لتغيرها الكثير وغيرها

ولكن يعيبها التفاعلية...

فإذا استطعنا الجمع بين مميزات ال Static مع تفاعلية ال Dynamic فهذا هو تحقيق المعادلة الصعبة وهذا ما قامت بة ال Static Page Generator وبالأخص Gatsby و مع تقنيات مثل React-Static

ويختلف هذا التعامل عن ال Server side App بأنة لايجود كود يعمل بالخادم بل يتم انشاء الكود في ناحية المستخدم وقت الطلب واظهارة في صورة ثابتة لا تتكون الا من HTML و CSS و JS مما يؤدي لسهولة فهرستها من قبل محركات البحث فضلا عن السرعة والأمان

تابعنا لتتعرف أكثر علي هذا التقنيات وكيفية استخدامها وعمل مشاريع بها مثل ال Blog و ال E-commerce وغيرها .....
